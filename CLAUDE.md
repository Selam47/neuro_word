📋 Standard Operating Procedures (SOP)
Analysis & Planning: Explain the task and proposed architecture BEFORE writing code.
The Clarification Gate: If data structures, UI, or libraries are ambiguous, ASK questions first.
Modular Execution (3-File Rule): If a task affects >3 files, break it into sub-tasks. No massive updates.
Resilience & Edge Cases: List potential errors, bottlenecks, and test scenarios for every update.
Root-Cause Debugging:
Create a reproduction scenario.
Explain the root cause (Design vs. Symptom).
Propose a permanent prevention plan.
